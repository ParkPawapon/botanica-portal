# Botanica Portal v0.0.1

This is the first foundation release for Botanica Portal.

## What Changed

- Added the production web application foundation with Next.js App Router, TypeScript, Tailwind CSS, MUI, Emotion, and Bun.
- Added routing skeletons for the landing page, plant registry, plant detail pages, and learning hub.
- Added centralized botanical design tokens, MUI theme configuration, global styles, reusable UI primitives, skeleton loading states, and SEO helpers.
- Added clean architecture boundaries for `app`, `features`, `entities`, `shared`, `config`, `theme`, `styles`, and `data`.
- Added CI quality gates, CodeQL, Dependency Review, release build workflow, branch and commit naming validation, CODEOWNERS, Dependabot, security policy, and contributor rules.
- Added Docker production build support and GitHub Container Registry publishing.

## Validation

- `bun run format:check`
- `bun run lint`
- `bun run type-check`
- `bun run build`
- `docker build -t botanica-portal:0.0.1 .`

## Versioning

This release uses `v0.0.1` because the project is still a foundation scaffold. It is intentionally not `v1.0.0`, which should be reserved for a stable production product with complete user-facing functionality.
